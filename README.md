# MogVault

A clean, ad-free WoW transmog outfit builder and farming-list generator.
Load your actual character model, build a transmog set, and get a focused
checklist showing exactly where each piece drops.

Non-commercial hobby project. No ads, no payments.

---

## Getting started

```bash
npm install
cp .env.example .env.local   # fill in your Blizzard API credentials
npm run dev
```

---

## Environment variables

Create `.env.local` (gitignored) from `.env.example`:

| Variable | Description |
|----------|-------------|
| `BLIZZARD_CLIENT_ID` | Client ID from [develop.battle.net](https://develop.battle.net/) |
| `BLIZZARD_CLIENT_SECRET` | Client secret from the same app registration |
| `BLIZZARD_REGION` | API region — `us`, `eu`, `kr`, or `tw` (default `us`) |

### Secret discipline

`BLIZZARD_CLIENT_SECRET` **must never be imported into a client component
or appear in the browser bundle.**

The OAuth client-credentials flow requires the secret to authenticate with
`https://oauth.battle.net/token`. All calls that touch the secret go through
Next.js Route Handlers (`app/api/**`), which run server-side only. The
browser talks only to your own `/api/*` routes — never directly to
`*.api.blizzard.com`.

If you ever see `BLIZZARD_CLIENT_SECRET` imported in a file that is not
inside `app/api/` or `lib/blizzard.ts`, that is a bug.

## Tech stack

| Layer | Choice |
|-------|--------|
| Framework | Next.js 16 (App Router, TypeScript) |
| Styling | Tailwind CSS v4 — void/purple theme |
| 3D viewer | `wow-model-viewer` (client-only, isolated in `components/CharacterViewer.tsx`) |
| Item / drop data | Blizzard Battle.net Game Data + Journal APIs |
| Character data | Blizzard Profile API |
| State | React local state + `localStorage` |
| Hosting | Vercel |

## Project structure

```
app/                    Next.js App Router pages and layouts
app/api/                Route Handlers — server-only, hold the Blizzard secret
components/
  CharacterViewer.tsx   Isolates all wow-model-viewer contact (client-only)
lib/
  blizzard.ts           Server-only Blizzard API client and OAuth token cache
scripts/
  build-source-index.ts One-off ingestion script (run manually after patches)
data/
  source-index.json     Pre-built item→source map (generated by the script above)
docs/
  step-0-findings.md    External dependency research notes
```

## Source index

The farming list is powered by `data/source-index.json`, a pre-built mapping
of `itemId → { instance, encounter, type }` derived from the Blizzard Journal
API. Re-generate it after major WoW patches:

```bash
npx tsx scripts/build-source-index.ts
```

Items not covered by the journal (world drops, vendor, crafted, quest, PvP)
will be absent from the index and labelled **Other source** in the farming list.
